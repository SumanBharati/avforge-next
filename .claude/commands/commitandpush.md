Commit all current changes and push to GitHub at the end of the work session.

Steps:
1. Run `git status` to see what files changed.
2. If there is nothing to commit, say so and stop.
3. Run `git add .` to stage all changes.
4. Use $ARGUMENTS as the commit message if provided. Otherwise, look at the changed files and write a short, clear commit message describing what was done.
5. Commit with that message.
6. Run `git push origin main`.
7. Confirm success and summarise what was pushed.
